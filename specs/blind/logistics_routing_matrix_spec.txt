Build one JSON AST module for a logistics routing matrix.

The module must be named demo.logistics-routing.

The program receives a collection named shipments. Each shipment has these fields:

- shipment-id: string
- region: string
- destination-zone: string
- weight-kg: number
- distance-km: number
- fragile: boolean
- promised-hours: number
- warehouse-stock: number
- carrier-email: string or missing

Create the main function route-plan that:

- accepts shipments
- returns a query over shipments as shipment
- keeps only shipments where promised-hours is less than 48 or fragile is true or weight-kg is greater than 25
- sorts by shipment.shipment-id ascending
- selects a map row with exactly these keys:
  - shipment-id from shipment.shipment-id
  - region from shipment.region
  - route is "express-fragile" when fragile is true and promised-hours is less than 24, "heavy-freight" when weight-kg is greater than 25, otherwise "standard-expedite"
  - priority is "same-day" when promised-hours is less than 24, "two-day" when promised-hours is less than 48, otherwise "normal"
  - stock-status is "stock-risk" when warehouse-stock is less than 5, otherwise "available"
  - contact is carrier-email when present, otherwise "dispatch-pool"

Do not include ordinary shipments outside the filter.
Do not invent fields.
