Build one JSON AST module for vendor payment risk operations.

The module must be named demo.vendor-payment-risk.

The program receives a collection named invoices. Each invoice has these fields:

- invoice-id: string
- vendor-id: string
- vendor-tier: string
- amount-due: number
- days-overdue: number
- dispute-count: number
- critical-vendor: boolean
- missing-tax-form: boolean
- owner-email: string or missing

Create the main function payment-risk-report that:

- accepts invoices
- returns a query over invoices as invoice
- keeps only invoices where days-overdue is greater than 10 or dispute-count is greater than 0 or missing-tax-form is true
- sorts by invoice.invoice-id ascending
- selects a map row with exactly these keys:
  - invoice-id from invoice.invoice-id
  - vendor-id from invoice.vendor-id
  - severity is "blocked-critical" when critical-vendor is true and missing-tax-form is true, "disputed" when dispute-count is greater than 0, "late" when days-overdue is greater than 10, otherwise "monitor"
  - payment-action is "hold" when missing-tax-form is true, "investigate" when dispute-count is greater than 0, otherwise "schedule"
  - amount-band is "large" when amount-due is greater than 50000, otherwise "standard"
  - owner is owner-email when present, otherwise "ap-queue"

Do not include clean invoices outside the filter.
Do not invent fields.
