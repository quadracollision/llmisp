Build a simple native Java Swing GUI for customer intake with local persistence.

The window title should be Customer Intake DB Test.

The GUI should have these main components:

- an applicant details form with fields full-name, email, phone, company, and requested-plan
- a qualification panel with fields budget-range, timeline, and current-provider
- a notes area for intake-notes
- a saved submissions table with columns submitted-at, full-name, email, company, requested-plan, and status
- action buttons for clear form, save draft, submit intake, and refresh submissions

Persistence requirements:

- Use a local SQLite database named customer-intake.sqlite3 in the project/output directory.
- Save draft should insert the current form values into a submissions table with status draft.
- Submit intake should insert the current form values into a submissions table with status submitted.
- Refresh submissions should reload the saved submissions table from the SQLite database.
- Clear form should clear user-entered fields only.

Safety requirements:

- Do not call external services.
- Do not use network actions.
- Do not use shell commands.
- Do not write arbitrary files other than the local SQLite database requested above.
- Database actions must be expressed as controlled GUI action declarations, not raw Java, raw Clojure, or handwritten event handler code.
